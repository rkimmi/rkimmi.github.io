Meditation Process

Tell your non-tech friend a story about a time you:

  • Got blocked on a simple problem
  • Solved a problem in an elegant way

In both cases:

What was the problem?

I spent too many hours on one problem in the Codecamp basic JavaScript tutorials in Sprint 4. The code made sense and I was confident that it should have worked, but nope.

What problem solving techniques did you use?

I avoided asking for help (which is not always a great strategy) since I was mostly certain of my answer. I brainstormed different ways to solve the problem but it was straightforward and so I kept returning to my original answer. The code recieved no specific syntax errors, but alas, it was missing just one comma.

How did you feel throughout the process?

Somewhat frustrated. I thought maybe I was being too hard-headed and felt annoyed by doing the same thing repeatedly.

This process became really funny after I had solved the problem though.

What did you learn?

Staying mostly calm is important, and taking a lot of breaks is helpful. I had told myself something like: 'if the problem is not solved in x amount of time then I can ask for help', which also relieved some pressure.

Reflect on how confident you feel using the problem solving techniques and process:

  • Pseudocode: Somewhat confident.
  • Trying something: Mostly confident.
  • Rubber ducky method: I love this idea.
  • Reading error messages: Somewhat confident.
  • Console.logging: Mostly confident.
  • Googling: Confident, although googling for code help is much harder than googling for general life help.
  • Asking your peers for help: I don't feel confident asking for help on the slack channel. I don't have much problem asking for help generally, it's just communicating online that can be stressful.
  • Asking coaches for help: More confident than asking peers, but would only do so as a last resort.
  • Improving your process with reflection: Thinking about writing these blogs is always more taxing than actually writing them. I do end up finding the reflections kind of fun when doing them.

Explain to your non-tech friend:

.map():

.map is used to create an array also assigned with a function. The results are that all things within the array will be influenced by the given function.

.filter():

.filter will process / filter out items in an array dependant on criteria that you give it.

.reduce():

.reduce can do a few things, some of which are:

  • Reduce items in an array down to a single value.
  • Flatten arrays; so arrays within arrays become just one larger array.
  • Count the instances of an item in an array, e.g, for the array: ['apples', 'watermelon', 'apples']; .reduce would return 'apples': 2, 'watermelon': 1.
