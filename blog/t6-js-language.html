JavaScript Language

In the JavaScript language, describe all the use cases for each of:

  • parentheses () - e.g. '1. Execute a function var result = doSomething()':

    • 1: Parenthesis () are used to define functions parameters, e.g;

    function aFunction (parameter1, possibly2) {

    };

    Sometimes functions have no parameters. In this case the parenthesis is left empty:

    function aFunction () {

    };

    2: Parathesis are additionally used when calling functions, which can look something like this:

    aFunction();

    3: We also see parathesis commonly when grouping expressions, e.g;

    (var i = 0; i < something; i++)

  • brackets []:

    • 1: Most commonly, brackets are used to both create and otherwise access arrays.

    E.g; to create an array: var array = ['here', 'is', 'a', 'long', 'list', 'of', 'array', 'items'] .

    To access values within this array, we can call the array using its variable name, 'array' and then within brackets specify which array item we want using its value. In this example, array[0] would return 'here', array[3] would return 'long', and array[7] would return the final item in the array, 'items'.

    2: Brackets are also used in bracket notations, which are formatted as brackets with a set of double quotes (" ") inside them. Bracket notation and dot notation are interchangeable; they do the same thing. Dot notation that looks something like this: object.property, would be written as: object["property"] using bracket notation.

  • braces {}:

    • 1: Braces define the beginnings and ends of a function, as well as delineates blocks within functions. These blocks include if and loop bodies. E.g;

    function aFunction () {

    for (expression) {

    if (expression) {}

    }

    else if {}

    }

    }

    2: Braces can be used to define arrays otherwise not defined within brackets. Here is an example using the .push function from our minesweeper.js file to push strings onto a previously empty array;

    newBoard.cells.push({row: noRows, col: noCols, isMine: hasMine, hidden: true});

  • single quotes '', and double quotes "":

    • 1: From my understanding, the use of single '', or double "" quotations is interchangeable. The use of one set of quotes alone can be tricky when using quotations within statements, however. In this case, you will want to use the other set of quotations that you did not use to open the statement, like this;

    'the message was written: "in double quotes!"'

    The use of double quotations is useful if the statement were to include any apostrophes (aka a single quote opening/closing statement) e.g;

    The statement: 'I've made a mistake!' would only return 'I', so "I've made a mistake!" is a smarter move.
